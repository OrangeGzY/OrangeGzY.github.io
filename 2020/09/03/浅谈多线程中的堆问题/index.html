浅谈多线程中的堆问题

首先感谢povcfe学长还有Shiki师傅的指导。之后准备专门写一篇malloc函数的文章

关于heap_info

在子线程的thread arena中一开始是没有heap区域的。当申请内存时（子线程调用malloc）就需要一个结构来记录对应的信息，而heap_info 的作用就是这个。而且当该heap的资源被使用完后，就必须得再次申请内存了。此外，一般申请的heap 是不连续的，因此需要记录不同heap之间的链接结构。

当主线程要求使用sbrk函数来保证堆空间是连续的时，那子线程只能使用mmap来分配堆空间。这样一来，由于mmap分配的特点，导致了线程分配的堆块是以块为单位的，如果某线程需要的堆块多的话，进程空间是不足的，再次使用mmap来分配heap时，二者并不连续，所以需要这样的数据结构来标识该块的所属和一些内存信息。

再说的简单点，就是用来记录子线程中不连续的heap信息的

A heap is a single contiguous memory region holding (coalesceable)
malloc_chunks. It is allocated with mmap() and always starts at an
address aligned to HEAP_MAX_SIZE.，这是官方对于heap的解释，当我们在子线程中需要拓展heap的时候，mmap会随机在对应的内存区域中找一个位置——这就代表了子线程的heap是不连续的，此时heap_info来记录每一个子线程（不连续的）堆段的信息。

该数据结构是专门为从 Memory Mapping Segment 处申请的内存准备的，即为非主线程准备的。

主线程可以通过 sbrk() 函数扩展 program break location 获得（直到触及Memory Mapping Segment），只有一个heap，没有 heap_info 数据结构。

该结构主要是描述堆的基本信息（子线程），包括

  • 堆对应的 arena 的地址
  • 由于一个线程申请一个堆之后，可能会使用完，之后就必须得再次申请。因此，一个可能会有多个堆。prev即记录了上一个 heap_info 的地址。这里可以看到每个堆的 heap_info 是通过单向链表进行链接的。
  • size 表示当前堆的大小
  • 最后一部分确保对齐
